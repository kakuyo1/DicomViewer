# 功能清单和优先级

## 1. 优先级定义

- `P0`：必须完成。没有这些内容，项目主线不成立。
- `P1`：强烈建议完成。能显著提升项目完成度与展示价值。
- `P2`：可选增强。时间充足时再做，不应影响主线交付。

当前版本的策略是：优先保证 `2D 完整稳定`，然后做 `MPR 可讲可演示`，最后补齐 `VR 基础能力`。

## 2. P0 功能

### 2.1 导入与数据构建

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 文件导入 | Open Folder | 从本地目录选择 DICOM 数据 |
| 扫描分组 | 按 SeriesInstanceUID 分组 | 支持单目录下多个 series |
| Series 选择 | 单 series 直载，多 series 弹窗选择 | 建立清晰导入流程 |
| 解析校验 | 基础元数据读取、排序、必要校验 | 确保能构建有效体数据 |
| 统一模型 | 构建 VolumeData | 供 Stack / MPR / VR 共享 |

### 2.2 Stack View

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 切片浏览 | 滚轮翻页 | 最基础的看片能力 |
| 视图操作 | Pan / Zoom | 2D 浏览核心交互 |
| 灰度调节 | Window / Level | 医学影像查看必备能力 |
| 基础测量 | Distance Measure | 只做距离测量 |
| 图像处理 | Invert | 用于基础演示 |
| 状态恢复 | Reset | 恢复默认显示状态 |
| 信息展示 | Overlay | 显示 slice、WW/WL、zoom、方向等 |

### 2.3 通用框架

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 主工作区 | WorkspaceWidget + QStackedWidget | 切换 Stack / MPR / VR 页面 |
| 视图封装 | SliceViewWidget | 统一 2D 切片交互与渲染承载 |
| 日志系统 | spdlog 基础接入 | 支持关键流程排查 |

## 3. P1 功能

### 3.1 Stack View 补全

| 模块 | 功能 | 说明 |

### 3.2 MPR View

| 模块 | 功能 | 说明 |

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 异步生成 | 后台线程生成 thumbnail | 避免主线程卡顿 |

## 4. P2 功能

### 4.1 VR View

| 模块 | 功能 | 说明 |
| --- | --- | --- |

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 异常处理 | 更完整的错误提示 | 提升稳定性和可演示性 |

## 5. 明确延期项

以下内容不建议在第一版周期内展开：

- 多序列同时显示
- Patient / Study / Series 树状管理
- DICOMDIR
- 压缩格式兼容
- 标注保存回写
- ROI、角度、面积、体积等高级测量
- 任意斜切与曲面重建
- 可编辑 transfer function
- 临床级精度校验

## 6. 依赖关系与开发顺序

建议按以下顺序推进：

1. Open Folder -> 扫描分组 -> series 选择
2. 选定 series -> 排序校验 -> 构建 VolumeData
3. SliceViewWidget 最小可用版本
4. Stack View 完整交互
5. 缩略图侧栏基础版
6. MPR 三正交显示
7. MPR 十字线与联动
8. VR 基础版
9. 性能优化与日志补全

## 7. 时间紧张时的裁剪建议

若 20-30 天周期出现压力，建议按以下顺序裁剪：

1. 先砍 `Flip Horizontal / Flip Vertical`
2. 简化 `Sync / Link`，可以默认常开，不单独做复杂开关逻辑
3. VR 只保留最基础 preset 和相机控制
4. 缩略图侧栏先做基础懒加载和缓存，不做复杂任务调度
5. overlay 信息保持精简，不额外扩展更多元数据显示
