# 项目范围说明

## 1. 项目定位

本项目是一个面向暑期实习面试展示的轻量化 DICOM Viewer，用于体现以下能力：

- C++/Qt 桌面应用开发能力
- 医学影像基础数据处理能力
- VTK 渲染管线设计能力
- 多线程加载与 UI 解耦能力
- 工程化分层与性能优化意识

该项目不是临床产品，也不追求真实医疗软件的完整性、兼容性和合规性。项目目标是在 20-30 天周期内，做出一个边界清晰、可演示、可讲透的作品集项目。

## 2. 第一版目标

第一版目标定义为：`2D 功能较完整 + 3D 初步可用`。

具体来说：

- 2D 部分做到稳定可演示，能够完整体现一套基础看片流程
- 3D 部分以 MPR 为主要亮点，Volume Rendering 作为辅助演示能力
- 架构上建立统一的 VolumeData 数据模型，支持 Stack / MPR / VR 复用
- 项目整体保持工程化分层，不过度追求功能堆叠

## 3. 目标用户与使用场景

目标用户不是临床医生，而是：

- 面试官
- 校招技术面试场景中的工程师
- 希望快速理解项目设计思路的评审者

典型使用场景：

- 从本地目录导入一组 CT DICOM 文件
- 选择一个可用的 series
- 在 Stack View 中完成 2D 浏览与基本操作
- 切换到 MPR View 演示三正交重建与联动
- 切换到 VR View 演示基础体渲染效果

## 4. 技术栈范围

- UI 与桌面交互：`C++ + Qt`
- 2D/3D 渲染：`VTK`
- DICOM 读取与解析：`DCMTK`
- 日志系统：`spdlog`
- 目标平台：`Linux`

## 5. 输入数据假设

为控制项目复杂度，第一版只支持以下输入范围：

- 本地目录导入
- 常见未压缩 CT DICOM 文件
- 单次只加载一个 series
- 数据足以构建规则体数据

程序在导入目录后，会先扫描并按 `SeriesInstanceUID` 分组：

- 若目录中仅有一个可用 series，则直接加载
- 若目录中有多个可用 series，则弹出 series 选择对话框，由用户选择其一进入主工作区

## 6. 本版本纳入范围

### 6.1 数据导入与构建

- 扫描目录中的 DICOM 文件
- 按 `SeriesInstanceUID` 分组
- 读取基础元数据并生成 series 摘要
- 对选中的 series 完成排序与基础校验
- 构建统一的 `VolumeData`

### 6.2 Stack View

- 单序列切片浏览
- 鼠标滚轮翻页
- 缩放
- 平移
- 窗宽/窗位调整
- 基础距离测量
- 水平翻转
- 垂直翻转
- 灰度反相
- Reset
- overlay 信息显示

### 6.3 MPR View

- 三正交视图：sagittal / coronal / axial
- 固定三窗格布局
- active view 概念
- 当前激活窗格上的缩放、平移、窗宽/窗位调整
- 基础距离测量
- 十字线显示开关 
- 三视图同步联动
- Reset

### 6.4 VR View

- 基础 Volume Rendering 演示
- Rotate / Pan / Zoom
- 三个硬编码 preset：Bone / Soft Tissue / Skin
- Reset Camera
- 在快速旋转时尽量保持流畅，必要时支持简单 LOD 思路

### 6.5 缩略图侧栏

- 右侧缩略图列表展示当前序列切片
- 点击缩略图跳转到对应切片
- 使用 Model/View 结构
- 懒加载
- 缓存
- 后台线程生成缩略图

### 6.6 日志与工程能力

- 关键流程日志输出
- 导入、解析、加载、渲染初始化等节点可观测
- 基本异常提示与失败路径处理

## 7. 明确不做的内容

以下内容明确不纳入第一版范围：

- Windows 或 macOS 适配
- PACS / 网络传输 / DICOM Query-Retrieve
- DICOMDIR 支持
- 压缩格式 DICOM 兼容
- 多 patient / study / series 管理界面
- 多序列同时显示
- 标注保存、回写、导出
- ROI、高级测量、角度、面积、体积统计
- 图像配准、分割、AI 模块
- 任意斜切 MPR
- 临床级精度保证与医疗合规能力

## 8. 非功能目标

第一版重点关注以下非功能目标：

- 交互流畅：2D 浏览、缩略图滚动、MPR 联动尽量避免明显卡顿
- 分层清晰：数据、视图、交互、后台任务职责明确
- 可演示：导入到浏览的主流程稳定
- 可讲述：核心设计可以被面试官快速理解

## 9. 完成标准

当以下条件满足时，可认为第一版达到预期：

- 可以成功导入本地 CT DICOM 目录并完成 series 选择
- 可以构建单个 `VolumeData` 并供 Stack / MPR / VR 共享
- Stack View 的核心 2D 交互完整可用
- MPR 三正交视图可正常显示并具备基础联动
- VR View 可以完成基础可视化演示
- 缩略图侧栏可滚动预览并支持点击跳转
- 程序在典型演示数据上稳定运行，适合面试现场展示
